# We 家計簿

二人（カップル / 夫婦）で共用する、純静的な家計簿アプリ。フレームワークもビルド工程も不要で、`index.html` をダブルクリックするだけでブラウザで動く。データは端末の `localStorage` に保存し、設定すれば **GitHub 上の非公開リポジトリの `data.json`** を介して二人の端末間で共有できる（準リアルタイム）。オフライン可・GitHub Pages 公開可。

> 公開リポジトリのシードはダミー（「太郎 / 花子」）。**本物の家計データは非公開リポジトリの `data.json` のみ**に置く。

## 設計方針（レッドライン・変更前に必読）

- ❌ バックエンド / DB を持たない — データは `localStorage` ＋ GitHub 上の単一 `data.json`（GitHub API 読み書き）。
- ❌ ビルドツールを使わない（バンドラ / コンパイル / フレームワーク CLI なし）— ダブルクリックで動く・オフライン可。
- 🔒 トークンはいかなるファイル / コードにも書かない・コミットしない（`localStorage` のみ）。
- 価値観：**無料・長く使える・低メンテ・シンプル**。

## 機能

- **概要** — 今月の残高、固定費・変動費・現金・入金 / 繰越金のまとめ、月リズムバー（月末が近づくと精算リマインダー）。
- **記帳** — 日々の支出を日付・分類ごとに記録（日付選択・現金支払いタグ可）。当月 / 未来月は変動費の大分類を追加・削除できる（翌月は既定5つに戻る）。
- **統計** — 月別総支出トレンド、分類内訳、過去平均との比較（直近6ヶ月 / 直近1年）。
- **履歴** — 月ごとの残高一覧（過去月も編集可）。
- **精算（下書き）** — 手元のお金を計算し、初期金額（＝今月の繰越金 ＋ 差額）を出して今月の繰越金に反映する補助ツール（下書きは自動保存）。
- **CSV 書き出し**（設定内・Excel 用、BOM 付きで日本語が文字化けしない）／ **JSON バックアップ**（右上）。
- 右上に「**最終更新**」時刻 ＋ 同期状態（同期済 / 同期中 / オフライン）を表示。
- **端末間同期**（任意・設定後）— 保存すると非公開リポジトリへ自動コミット、開く / フォーカス時に相手の更新を取得。

## ローカルでの動かし方 / 本地运行

最も簡単なのは `index.html`（`css/` と `js/` が**隣にある**フォルダ内のもの）をブラウザで直接開く（ダブルクリック）。
ローカルファイル制限が厳しいブラウザでは静的サーバーを立てると確実：

```bash
python -m http.server 8000   # または: npx serve
```

→ ブラウザで `http://localhost:8000` を開く。

## ディレクトリ構成

```
Accounting/                # ← 公開リポジトリ（コードのみ・全てダミー）
├─ index.html              # 骨格 + 6 つの <script> + マウント先 DOM
├─ css/
│  └─ styles.css           # 全スタイル（原生 CSS、暖色ミニマル）
├─ js/
│  ├─ data.js              # 最小シード（ダミー「太郎/花子」・空の当月のみ）
│  ├─ storage.js           # 保存 / 読込（localStorage、失敗時メモリ）
│  ├─ compute.js           # 純粋な会計計算（残高・固定費・変動費 …）
│  ├─ ui.js                # 描画・操作（原生アニメ、ライブラリなし）
│  ├─ main.js              # ルーティング + 初期化 + イベント束ね
│  └─ sync.js              # GitHub 同期レイヤー（ローカル優先・非同期）
├─ README.md
└─ .gitignore              # data.json / *.token / .env を除外

（別リポジトリ・非公開）
└─ data.json               # 本物の家計データ — 公開フォルダには絶対に置かない
```

読み込み順：`data → storage → compute → ui → main → sync`（通常の `<script>`、同一グローバルスコープ）。

## 会計の考え方（コア不変量）

- 月末残高 = 繰越金 + 入金合計 − 固定費合計 − 変動費合計。
- 保証料（+ 引落手数料）= 引き落とし総額 − 賃料 − 管理費 − 電気（残差として算出）。

## 端末間同期のしくみ（任意）

- **ローカル優先**：起動時はまずローカルを即表示 → 裏で非公開リポジトリから取得し、新しければ反映。
- **保存**：まずローカル → 1.5 秒のデバウンス後に非公開リポジトリへコミット（push）。
- **取得**：ウィンドウのフォーカス / 表示 / ネット復帰で自動取得（pull）。準リアルタイム（コミット約 1–3 秒）。
- **競合**：月単位の 3-way マージ。**別々の月の編集は互いに上書きしない**；同じ月を同時に編集したときのみ、より新しい保存（`lastModified`）を採用。
- **設定**：アプリの「設定」→ 下部「同期設定」に GitHub ユーザー名 / 非公開リポジトリ名 / パス（`data.json`）/ トークンを入力（各端末で一度）。トークンは `localStorage` のみ、コードには入らない。

## デプロイ（GitHub Pages）＋ 同期セットアップ

1. **公開リポジトリ**を作り、`Accounting/` の中身（コード一式）を上げる。
2. **非公開リポジトリ**を作り、`data.json` を**ルート（直下）**に置く。
3. **細粒度 PAT** を発行：その**非公開データリポジトリの Contents 読み書き**のみ許可（最小権限）。
4. 公開リポジトリの Settings → Pages で公開ブランチ（例: `main`）を指定 → 端末からアクセスできる URL を得る。
5. **二台とも**その URL を開き、「設定」→「同期設定」にユーザー名 / 非公開リポジトリ名 / パス（`data.json`）/ トークンを各端末で一度だけ入力。

## データの取り扱い（重要）

- 本物の家計データは**非公開リポジトリの `data.json` のみ**。公開フォルダ（`Accounting/`）には絶対に置かない（`.gitignore` で `data.json` を除外済み）。
- 公開リポジトリの `data.js` はダミーの最小シード（本物の数字なし）。
- 開発ログ `PROJECT_JOURNAL.md` は本物の数値・基準を含むため**非公開 / ローカルのみ**に置く。
- トークンは決してファイル / コードに書かない・コミットしない（`localStorage` のみ）。
